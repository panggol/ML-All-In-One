# 06_data_management_requirements.md

## 模块概述
**模块名称**: 数据管理 (Data Management)
**模块类型**: 功能模块
**优先级**: P1（核心模块）
**预估工作量**: 3-4 小时

## 目标
提供一个统一的数据管理中心，支持数据文件的上传、浏览、预览、统计信息查看和数据导出功能。

## 用户故事
作为 ML 平台用户，我需要在训练模型之前：
1. 上传和浏览数据集
2. 预览数据集结构和内容
3. 查看数据集的统计信息（行数、列数、数据类型、缺失值等）
4. 删除不需要的数据集
5. 导出数据集为 CSV 格式

## 功能需求

### 1. 数据文件列表
- **展示所有已上传的数据文件**
  - 显示：文件名、文件大小、上传时间、行数、列数
  - 支持按上传时间排序（默认降序）
  - 显示数据集缩略信息

### 2. 数据上传
- **支持拖拽上传**
  - 支持 CSV 格式
  - 显示上传进度
  - 上传成功后自动刷新列表
  - 显示上传成功提示

### 3. 数据预览
- **查看数据集前 10 行数据**
  - 以表格形式展示
  - 显示列名
  - 识别并显示数据类型

### 4. 数据统计
- **显示详细统计信息**
  - 总行数、总列数
  - 每列的数据类型
  - 每列的缺失值数量
  - 每列的唯一值数量（分类列）
  - 每列的数值范围（数值列：最小值、最大值、平均值）

### 5. 数据导出
- **导出数据集**
  - 支持下载为 CSV 格式
  - 保持原数据格式

### 6. 数据删除
- **删除不需要的数据集**
  - 显示确认对话框
  - 删除后自动刷新列表

## 交互设计

### 上传交互
1. 用户点击上传按钮或拖拽文件到拖拽区域
2. 显示上传进度条
3. 上传完成后显示成功提示，自动刷新列表
4. 如果上传失败，显示错误信息

### 预览交互
1. 用户点击某数据集的「预览」按钮
2. 展开详情面板，显示该数据集的前 10 行
3. 表格中支持横向滚动查看所有列

### 统计交互
1. 用户点击某数据集的「统计」按钮
2. 展开详情面板，显示该数据集的统计信息
3. 数值列显示：数量、均值、标准差、最小值、Q1、中位数、Q3、最大值
4. 分类列显示：数量、唯一值数量、最常见的5个值

### 删除交互
1. 用户点击某数据集的「删除」按钮
2. 弹出确认对话框：「确定删除 xxx.csv 吗？此操作不可撤销。」
3. 用户确认后执行删除，列表自动刷新
4. 显示删除成功提示

## API 需求

| 操作 | API Endpoint | Method | 说明 |
|------|-------------|--------|------|
| 列表 | `/data/list` | GET | 获取所有数据集列表 |
| 上传 | `/data/upload` | POST | 上传 CSV 文件 |
| 删除 | `/data/{id}` | DELETE | 删除指定数据集 |
| 预览 | `/data/{id}/preview` | GET | 获取数据集前 N 行 |
| 统计 | `/data/{id}/stats` | GET | 获取数据集统计信息 |

### 响应数据结构

```typescript
// 数据文件
interface DataFile {
  id: number
  filename: string
  size: number      // bytes
  rows: number
  columns: string[]
  created_at: string
}

// 预览响应
interface PreviewResponse {
  rows: unknown[][]
  columns: string[]
  total_rows: number
}

// 统计响应
interface StatsResponse {
  total_rows: number
  total_columns: number
  column_stats: {
    column: string
    dtype: string
    null_count: number
    unique_count?: number
    min?: number
    max?: number
    mean?: number
    std?: number
    top_values?: { value: string; count: number }[]
  }[]
}
```

## 技术实现

### 前端
- 使用 React + TypeScript
- 复用现有组件：Card, Button, Input
- 表格使用自定义表格（可参考 Inference.tsx）
